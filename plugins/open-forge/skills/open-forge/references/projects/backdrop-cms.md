# Backdrop CMS

Full-featured PHP content management system for small-to-medium websites and non-profits. A community fork of Drupal 7 that keeps Drupal's power while reducing complexity. Supports blogs, galleries, intranets, e-commerce, community sites, and multilingual content. Upstream: <https://github.com/backdrop/backdrop> — GPL-2.0.

## Compatible install methods

| Method | Upstream | First-party? | When to use |
|---|---|---|---|
| Manual PHP install | <https://backdropcms.org/installation> | Yes | LAMP/LEMP stack. PHP 7.4+ + MySQL/MariaDB. |
| Docker Compose | Community | Community | Containerised deployment. Uses official `backdrop/backdrop` Docker Hub image. |
| Softaculous / cPanel auto-install | Various hosts | Community | Shared hosting auto-installer. No server access needed. |

## Docker Compose method

```yaml
version: "3.8"

services:
  backdrop-db:
    image: mysql:8.0
    container_name: backdrop-db
    restart: unless-stopped
    environment:
      MYSQL_ROOT_PASSWORD: REPLACE_ROOT_PASSWORD
      MYSQL_DATABASE: backdrop
      MYSQL_USER: backdrop
      MYSQL_PASSWORD: REPLACE_DB_PASSWORD
    volumes:
      - backdrop_db:/var/lib/mysql

  backdrop:
    image: backdrop/backdrop:latest
    container_name: backdrop
    restart: unless-stopped
    depends_on:
      - backdrop-db
    ports:
      - "8080:80"
    environment:
      BACKDROP_DB_HOST: backdrop-db
      BACKDROP_DB_USER: backdrop
      BACKDROP_DB_PASSWORD: REPLACE_DB_PASSWORD
      BACKDROP_DB_NAME: backdrop
    volumes:
      - backdrop_files:/var/www/html/files
      - backdrop_modules:/var/www/html/modules
      - backdrop_themes:/var/www/html/themes

volumes:
  backdrop_db:
  backdrop_files:
  backdrop_modules:
  backdrop_themes:
```

After starting, navigate to `http://<host>:8080/core/install.php` to complete the web installer.

## Manual PHP method

Requirements: PHP 7.4+ (8.x recommended), MySQL 5.5+ or MariaDB, Apache/Nginx, 50 MB disk space.

```bash
# Download latest release
wget https://github.com/backdrop/backdrop/releases/latest/download/backdrop.zip
unzip backdrop.zip -d /var/www/html/

# Set permissions
chown -R www-data:www-data /var/www/html/backdrop
chmod -R 755 /var/www/html/backdrop
chmod -R 777 /var/www/html/backdrop/files /var/www/html/backdrop/tmp

# Create MySQL database
mysql -u root -p -e "CREATE DATABASE backdrop; GRANT ALL ON backdrop.* TO 'backdrop'@'localhost' IDENTIFIED BY 'REPLACE_DB_PASSWORD';"

# Visit http://<host>/backdrop/core/install.php to run the web installer
```

Apache VirtualHost `.htaccess` is bundled. For Nginx, see the upstream Nginx config example at <https://backdropcms.org/guide/nginx>.

## File structure

| Path | Contents |
|---|---|
| `/files/` | Uploaded files, images, assets — must be writable |
| `/modules/` | Contributed modules directory |
| `/themes/` | Contributed themes directory |
| `/core/` | Backdrop core (don't modify) |
| `settings.php` | Database and site config (auto-generated by installer) |

## Upgrade procedure

### Minor/patch updates (1.x.y)

```bash
# Via admin UI: Admin > Reports > Available updates
# Or manually:
cd /var/www/html/backdrop
wget https://github.com/backdrop/backdrop/releases/latest/download/backdrop.zip
unzip -o backdrop.zip
# Visit http://<host>/core/update.php to run DB updates
```

### Docker upgrade

```bash
docker compose pull backdrop
docker compose up -d
# Visit http://<host>/core/update.php after upgrade
```

## Module and theme management

Backdrop uses a layout-first approach (borrowed from Drupal). Modules extend functionality; themes control appearance. Install contributed modules at `/modules/`, themes at `/themes/`.

Backdrop-contrib modules: <https://github.com/backdrop-contrib>

```bash
# Install a module
drush dl <module-name>
drush en <module-name>
# Or via admin: Admin > Modules
```

## Gotchas

- **Backdrop ≠ Drupal.** While forked from Drupal 7, Backdrop modules are NOT Drupal 8/9/10 compatible. Backdrop has its own contrib ecosystem at backdrop-contrib.
- **Files directory must be writable.** `/files/` holds all user uploads. If it's not writable by the web server, uploads fail and the site will throw errors.
- **settings.php is auto-generated.** Don't create it manually — let the installer do it. For Docker, it's auto-configured via env vars.
- **Update.php is required after upgrades.** Always visit `/core/update.php` after uploading new code to run database schema updates.
- **URL rewriting.** Clean URLs require `mod_rewrite` on Apache or equivalent Nginx config. The bundled `.htaccess` handles Apache automatically.
- **Cron.** Backdrop uses built-in cron (`/core/cron.php`) for maintenance tasks (cache clear, update checks). Configure a system cron job or use the "Poor man's cron" module.
