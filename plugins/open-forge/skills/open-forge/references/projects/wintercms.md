# WinterCMS

WinterCMS is a free, open-source CMS built on the Laravel PHP framework. It was forked from October CMS in March 2021 due to differences in open-source governance. WinterCMS focuses on developer-friendliness, security, and simplicity, with an active plugin/theme ecosystem.

Official site: https://wintercms.com/  
Source: https://github.com/wintercms/winter  
Docs: https://wintercms.com/docs  
Latest release: v1.2.12 (February 2026)

## Compatible Combos

| Infra | Runtime | Notes |
|-------|---------|-------|
| Any Linux VM / VPS | PHP 8.1+ + MySQL/PostgreSQL/SQLite | Recommended; Composer-based install |
| Any Linux VM / VPS | Docker (community images) | No official Docker image; community Dockerfiles exist |
| Shared hosting | PHP 8.1+ | Works on standard LAMP/LEMP stacks |

## Inputs to Collect

**Phase: Planning**
- Domain/hostname
- Database type: MySQL, PostgreSQL, or SQLite
- Database credentials (host, port, name, user, password) if not SQLite
- PHP version (8.1+ required)

**Phase: Installation**
- Admin account credentials (auto-generated by `winter:up`, displayed in terminal)
- Application URL (`APP_URL`)
- App key (auto-generated)

## Software-Layer Concerns

**Quick install via Composer:**
```bash
composer create-project wintercms/winter /var/www/example.com "dev-develop"
cd /var/www/example.com
php artisan winter:env        # generates .env with guided prompts
php artisan winter:up         # runs migrations + creates admin account
```

**Key `.env` variables:**
```env
APP_NAME=WinterCMS
APP_URL=https://example.com
APP_KEY=                      # auto-generated

DB_CONNECTION=mysql
DB_HOST=127.0.0.1
DB_PORT=3306
DB_DATABASE=wintercms
DB_USERNAME=wintercms
DB_PASSWORD=CHANGE_ME
```

**Directory structure:**
- Config: `config/`
- Plugins: `plugins/`
- Themes: `themes/`
- Storage/uploads: `storage/`
- Public web root: `public/` (or project root with `.htaccess`)

**Web server config:**
- Apache: `.htaccess` included; enable `mod_rewrite`
- Nginx: requires `try_files $uri $uri/ /index.php?$query_string;`

**Permissions:**
```bash
chown -R www-data:www-data storage/ plugins/ themes/
chmod -R 775 storage/ plugins/ themes/
```

## Upgrade Procedure

1. Back up database and `storage/` directory
2. Update dependencies: `composer update`
3. Run migrations: `php artisan winter:up`
4. Clear caches: `php artisan cache:clear && php artisan config:clear`
5. Check changelog: https://github.com/wintercms/winter/blob/develop/CHANGELOG.md

## Gotchas

- **Fork of October CMS**: WinterCMS and October CMS share much of their API/plugin ecosystem; many October plugins work with WinterCMS but check compatibility
- **No official Docker image**: Must use community-maintained images or build your own
- **PHP requirements**: Requires PHP 8.1+ (PHP 8.2+ recommended); older PHP not supported in recent releases
- **`dev-develop` branch**: The quick-start command installs the development branch; for production, pin to a stable version tag
- **Plugin marketplace**: WinterCMS Marketplace at https://wintercms.com/marketplace — separate from October CMS marketplace
- **SQLite support**: Fine for development/small sites; not recommended for production at scale
