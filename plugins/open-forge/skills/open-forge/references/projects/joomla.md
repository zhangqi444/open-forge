# Joomla!

Open-source PHP content management system powering ~2% of all websites. Supports MySQL, MariaDB, and PostgreSQL. Includes a rich extension ecosystem (templates, components, modules, plugins) via the Joomla Extensions Directory. Current stable: Joomla! 5.x. Upstream: <https://github.com/joomla/joomla-cms>. Docs: <https://docs.joomla.org>.

## Compatible combos

| Infra | Runtime | Notes |
|---|---|---|
| VPS / bare metal | Apache + PHP + MySQL/MariaDB | Classic LAMP stack; upstream-recommended |
| VPS / bare metal | NGINX + PHP-FPM + MySQL/MariaDB | LEMP stack; needs manual NGINX config |
| VPS / bare metal | Docker Compose | Official Docker image available |
| Shared hosting | PHP + MySQL | Joomla runs on most PHP shared hosts |

## Inputs to collect

| Phase | Prompt | Notes |
|---|---|---|
| preflight | "Docker or traditional LAMP?" | Drives install path |
| db | "Database name, user, password?" | Create dedicated DB user; do not use root |
| db | "MySQL/MariaDB or PostgreSQL?" | MySQL/MariaDB recommended; PostgreSQL supported |
| site | "Site name?" | Shown in browser title and admin panel |
| admin | "Admin username, password, and email?" | First superuser account |
| domain | "Public domain?" | Used for SEF URLs and TLS setup |

## Software-layer concerns

- Config file: configuration.php in Joomla root (generated by web installer; do not edit manually in most cases)
- Default port: 80/443 (web server)
- Data dirs: images/ (media uploads), administrator/logs/, tmp/ (temp files)
- PHP requirements: PHP 8.1+ (Joomla 5.x); extensions: json, simplexml, gd, mysqli/pdo_mysql, zip, xml, mbstring
- Extensions: install via Extensions > Install in admin; JED (https://extensions.joomla.org) is the official marketplace
- Cron: required for task scheduler (Joomla 4+); add a cron entry for /path/to/joomla/cli/joomla.php task:run

### Docker Compose

```yaml
services:
  joomla:
    image: joomla:latest
    depends_on:
      - mysql
    environment:
      JOOMLA_DB_HOST: mysql
      JOOMLA_DB_USER: joomla
      JOOMLA_DB_PASSWORD: <db-password>
      JOOMLA_DB_NAME: joomla
    ports:
      - "80:80"
    volumes:
      - joomla-data:/var/www/html
    restart: unless-stopped

  mysql:
    image: mysql:8.0
    environment:
      MYSQL_ROOT_PASSWORD: <root-password>
      MYSQL_DATABASE: joomla
      MYSQL_USER: joomla
      MYSQL_PASSWORD: <db-password>
    volumes:
      - mysql-data:/var/lib/mysql
    restart: unless-stopped

volumes:
  joomla-data:
  mysql-data:
```

Open http://localhost to run the web installer on first access.

### LAMP quick-install (Ubuntu 22.04)

```bash
# Install stack
sudo apt-get install apache2 php8.1 php8.1-{mysql,xml,gd,zip,mbstring,curl} mysql-server

# Download Joomla
curl -L https://downloads.joomla.org/latest -o joomla.zip
sudo unzip joomla.zip -d /var/www/html/joomla
sudo chown -R www-data:www-data /var/www/html/joomla

# Browse to http://your-server/joomla to run installer
```

## Upgrade procedure

1. Always backup database + files first
2. Use Joomla's built-in updater: Admin > System > Update > Joomla (one-click)
3. For major versions (4.x → 5.x): follow the official migration guide at https://docs.joomla.org/Migration
4. Docker: pull new image, copy files to volume, run database update script from admin

## Gotchas

- **File permissions**: Joomla needs write access to tmp/, logs/, and images/. Set ownership to web server user (www-data or apache). Too-permissive permissions (777) are a security risk.
- **configuration.php security**: Should not be web-accessible for writing after install. Joomla auto-protects it; confirm with a 403 test.
- **Extensions from untrusted sources**: Joomla extensions run as PHP on your server. Only install from the JED or trusted vendors.
- **Cron for task scheduler**: Without a cron job, Joomla 4+ features (cache cleanup, update checks, email queue) won't run automatically.
- **SEF URLs**: Enable .htaccess-based SEF URLs (mod_rewrite) in Admin > Global Config > SEO. Requires Apache mod_rewrite or NGINX equivalent.
- **PHP memory**: Increase memory_limit to 256M+ in php.ini for sites with many extensions or large media libraries.
- **Docker image from source**: The GitHub repo is the source; use https://downloads.joomla.org for installable packages or the `joomla` Docker Hub image.
