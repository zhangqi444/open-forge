# Ente

End-to-end encrypted photo storage and 2FA backup platform. Ente Photos is an open-source alternative to Google Photos; Ente Auth is an open-source alternative to Authy. Both apps share the same self-hosted server (museum). All data is encrypted client-side before upload — the server never sees plaintext content.

**Official site:** https://ente.com  
**Source:** https://github.com/ente-io/ente  
**Upstream docs:** https://ente.com/help/self-hosting/  
**Quickstart guide:** https://github.com/ente-io/ente/blob/main/server/docs/quickstart.md

## Inputs to Collect

### All phases
| Variable | Description | Example |
|----------|-------------|---------|
| `POSTGRES_PASSWORD` | PostgreSQL password | auto-generated by quickstart |
| `MINIO_USER` | MinIO access key | auto-generated by quickstart |
| `MINIO_PASSWORD` | MinIO secret key | auto-generated by quickstart |
| `MUSEUM_KEY_ENCRYPTION` | 32-byte base64 encryption key | auto-generated by quickstart |
| `MUSEUM_KEY_HASH` | 64-byte base64 hash key | auto-generated by quickstart |
| `MUSEUM_JWT_SECRET` | URL-safe base64 JWT secret | auto-generated by quickstart |

### Optional — public URL overrides (museum.yaml)
| Key | Description | Default |
|-----|-------------|---------|
| `apps.public-albums` | URL for public album sharing | `http://localhost:3002` |
| `apps.accounts` | URL for passkey 2FA | `http://localhost:3001` |

## Software-Layer Concerns

### Quickstart (recommended)
```sh
sh -c "$(curl -fsSL https://raw.githubusercontent.com/ente-io/ente/main/server/quickstart.sh)"
# Creates my-ente/ directory with compose.yaml + museum.yaml, then starts cluster
cd my-ente && docker compose up -d
```

Requirements: Docker Compose >= 2.30, `base64` command available.

### Services started by quickstart

| Service | Port | Description |
|---------|------|-------------|
| museum (API) | `8080` | Ente server — handles auth, metadata, storage proxy |
| web (Photos) | `3000` | Ente Photos web app |
| web (public albums) | `3002` | Public album sharing app |
| postgres | internal | Database |
| minio | `3200` | Local S3-compatible object storage |
| socat | — | Bridges museum → minio on localhost:3200 |

### Docker images
- API server: `ghcr.io/ente-io/server`
- Web app: `ghcr.io/ente-io/web`

### Config file (museum.yaml)
Generated automatically by quickstart. Key sections:
```yaml
db:
  host: postgres
  port: 5432
  name: ente_db
  user: pguser
  password: <generated>

s3:
  are_local_buckets: true
  use_path_style_urls: true
  b2-eu-cen:
    key: <minio-user>
    secret: <minio-pass>
    endpoint: localhost:3200
    region: eu-central-2
    bucket: b2-eu-cen

key:
  encryption: <32-byte-base64>
  hash: <64-byte-base64>

jwt:
  secret: <url-safe-base64>

apps:
  public-albums: http://localhost:3002
  accounts: http://localhost:3001
```

### Data directories
- `museum.yaml` — server config (keep secret; contains encryption keys)
- `data/` — local object storage (MinIO buckets); back this up
- PostgreSQL volume — database; back this up

### Account creation
The first account is created via the Ente mobile/desktop/web app pointed at your server. During signup, a verification OTP is printed to museum logs (not emailed) by default:
```sh
docker compose logs museum | grep "verification"
```
See: https://ente.com/help/self-hosting/faq/otp

## Upgrade Procedure

1. Pull latest images: `docker compose pull`
2. Recreate: `docker compose up -d`
3. Check release notes: https://github.com/ente-io/ente/releases
4. The `museum.yaml` config format rarely changes; review diff if upgrading across major versions

## Gotchas

- **E2E encryption means no server-side recovery** — if encryption keys in `museum.yaml` are lost, encrypted data is unrecoverable. Back up `museum.yaml` securely.
- **Docker Compose >= 2.30 required** — the quickstart script enforces this; older versions may fail silently
- **Verification code in logs, not email** — on a fresh self-hosted instance, OTPs for account creation appear in `museum` container logs, not email (unless SMTP is configured)
- **MinIO is local-only by default** — the quickstart uses MinIO as local object storage; for production, configure an external S3-compatible bucket in `museum.yaml`
- **socat bridge** — the `socat` sidecar is required so museum can reach minio at `localhost:3200` via `network_mode: service:museum`; don't remove it in the default setup
- **Both Photos and Auth use same server** — one museum instance serves both Ente Photos and Ente Auth; clients share credentials
