# Ackify

> Self-hosted document read-acknowledgement platform ŌĆö prove that collaborators have read and signed important documents using Ed25519 cryptographic signatures with an immutable audit trail.

**URL:** https://www.ackify.eu
**Source:** https://github.com/kolapsis/ackify
**License:** AGPL-3.0

## Compatible Combos

| Infra | Runtime | Notes |
|-------|---------|-------|
| Any   | Docker Compose | Official image: `kolapsis/ackify`; requires PostgreSQL 16 |
| Any   | Build from source | Go 1.24.5 backend + Vue 3 / TypeScript frontend |

## Inputs to Collect

### Provision phase
- Domain / public URL (e.g. `sign.your-domain.com`)
- At least ONE authentication method: OAuth2 credentials (Google, GitHub, GitLab, or custom) **or** SMTP server for MagicLink (passwordless email)
- PostgreSQL 16 instance (included in compose)

### Deploy phase
- `APP_DNS` ŌĆö public domain
- `ACKIFY_BASE_URL` ŌĆö full public URL (e.g. `https://sign.your-domain.com`)
- `ACKIFY_ORGANISATION` ŌĆö display name for your organisation
- `POSTGRES_USER` / `POSTGRES_PASSWORD` / `POSTGRES_DB` ŌĆö database credentials
- `ACKIFY_OAUTH_COOKIE_SECRET` ŌĆö AES-256 session encryption key (generate with `openssl rand -base64 32`)
- `ACKIFY_ED25519_PRIVATE_KEY` ŌĆö Ed25519 private key for signing (auto-generated by install script)
- `ACKIFY_ADMIN_EMAILS` ŌĆö comma-separated admin email addresses
- OAuth2 (optional): `ACKIFY_OAUTH_PROVIDER`, `ACKIFY_OAUTH_CLIENT_ID`, `ACKIFY_OAUTH_CLIENT_SECRET`
- MagicLink SMTP (optional): `ACKIFY_MAIL_HOST`, `ACKIFY_MAIL_PORT`, `ACKIFY_MAIL_USERNAME`, `ACKIFY_MAIL_PASSWORD`, `ACKIFY_MAIL_FROM`

## Software-layer Concerns

### Docker Compose
```yaml
# Use the automated install script (recommended):
bash <(curl -fsSL https://raw.githubusercontent.com/kolapsis/ackify/main/install/install.sh)
cd ackify-ce
nano .env   # add OAuth2 / SMTP credentials
docker compose up -d
```

Core services in `compose.yml`:
```yaml
services:
  ackify-migrate:           # runs DB migrations, then exits
    image: kolapsis/ackify
    command: ["/app/migrate", "up"]
    depends_on:
      ackify-db:
        condition: service_healthy

  ackify-ce:                # main app, port 8080
    image: kolapsis/ackify
    restart: unless-stopped
    ports:
      - "8080:8080"
    environment:
      ACKIFY_BASE_URL: "${ACKIFY_BASE_URL}"
      ACKIFY_DB_DSN: "postgres://ackify_app:${ACKIFY_APP_PASSWORD}@ackify-db:5432/ackify?sslmode=disable"
      # ... (see .env for full variable list)

  ackify-db:                # PostgreSQL 16
    image: postgres:16-alpine
    volumes:
      - ackify_data:/var/lib/postgresql/data

  backup:                   # daily automated backup via docker-volume-backup
    image: offen/docker-volume-backup:v2.46.1
    environment:
      BACKUP_CRON_EXPRESSION: "@daily"
      BACKUP_RETENTION_DAYS: 7
```

### Config / env vars
- `ACKIFY_BASE_URL`: full public URL of the instance
- `ACKIFY_ORGANISATION`: organisation display name
- `ACKIFY_ORGANISATION_DOMAIN`: restrict sign-ups to this email domain (optional)
- `ACKIFY_DB_DSN`: PostgreSQL connection string
- `ACKIFY_OAUTH_COOKIE_SECRET`: AES-256 session key (required)
- `ACKIFY_ED25519_PRIVATE_KEY`: Ed25519 signing key (required; auto-generated by install script)
- `ACKIFY_ADMIN_EMAILS`: comma-separated admin email list
- `ACKIFY_OAUTH_PROVIDER`: `google`, `github`, `gitlab`, or `custom`
- `ACKIFY_OAUTH_CLIENT_ID` / `ACKIFY_OAUTH_CLIENT_SECRET`: OAuth2 app credentials
- `ACKIFY_AUTH_MAGICLINK_ENABLED`: set `true` to enable passwordless email auth
- `ACKIFY_MAIL_HOST` / `ACKIFY_MAIL_PORT` / `ACKIFY_MAIL_USERNAME` / `ACKIFY_MAIL_PASSWORD` / `ACKIFY_MAIL_FROM`: SMTP config for MagicLink and reminders
- `ACKIFY_STORAGE_TYPE`: `local` or `s3` for document uploads (optional)
- `ACKIFY_STORAGE_LOCAL_PATH`: local storage path (default `/data/documents`)
- `ACKIFY_STORAGE_MAX_SIZE_MB`: max document upload size (default `50`)
- `ACKIFY_TELEMETRY`: set `true` to enable telemetry (default `false`)

### Data dirs
- `ackify_data` (named volume) ŌĆö PostgreSQL data
- `/data/documents` ŌĆö uploaded documents (if local storage enabled)
- `/data/telemetry` ŌĆö telemetry data (if enabled)

## Upgrade Procedure
```bash
docker compose pull
docker compose up -d
```
The `ackify-migrate` service runs automatically on container start and applies any pending DB migrations.

## Gotchas
- **Authentication is required** ŌĆö at least one auth method (OAuth2 or MagicLink) must be configured before the app will work; it will not start without credentials.
- **OAuth auto-detection** ŌĆö OAuth2 is enabled automatically when `ACKIFY_OAUTH_CLIENT_ID` and `ACKIFY_OAUTH_CLIENT_SECRET` are set; MagicLink requires `ACKIFY_AUTH_MAGICLINK_ENABLED=true`.
- The install script auto-generates `ACKIFY_OAUTH_COOKIE_SECRET` and `POSTGRES_PASSWORD`; manual installs must generate these manually (`openssl rand -base64 32`).
- The `ackify-migrate` container must complete successfully before `ackify-ce` can serve traffic; `depends_on: condition: service_healthy` handles this in the compose file.
- The built-in backup service uses `docker-volume-backup` and saves to `./backup/` by default; configure S3 for off-site backups.
- Rate limits: 5 auth requests/min, 100 general requests/min.
- Image is distroless and < 30 MB.
