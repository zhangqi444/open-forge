# 015

**Temporary file sharing platform** — self-hosted one-time/temporary file and text upload, processing, and sharing. Named after Ichigo from *DARLING in the FRANXX*. Built with Vue 3 + Nuxt 3 (frontend) and Go (backend), using Redis + Asynq for async task processing.

**Official demo:** https://send.fudaoyuan.icu  
**Source:** https://github.com/keven1024/015  
**License:** AGPL-3.0

## Inputs to Collect

### Provision phase
| Input | Description | Default |
|-------|-------------|---------|
| `HTTP_PORT` | External port | `8080` |

### Config file (`config.yaml`)
| Key | Description | Default / Required |
|-----|-------------|-------------------|
| `share.download_secret` | Secret for JWT download tokens | **required** |
| `share.download_window` | Download token validity window (hours) | `12` |
| `share.password_salt` | Salt for password protection | **required** |
| `upload.path` | File storage path inside container | `/upload` |
| `upload.maximum` | Max total storage (e.g. `100GiB`) | `100GiB` |
| `redis.url` | Redis connection URL | `redis://redis:6379/0` |
| `site.url` | Public URL of this instance | **required** |
| `features.*` | Enable/disable file-share, text-share, image-compress, image-convert | all `true` |

## Software-layer Concerns

### Docker Compose
```yaml
services:
  app:
    image: fudaoyuanicu/015-app:latest
    container_name: 015-app
    volumes:
      - ./uploads:/uploads
      - ./config.yaml:/app/config.yaml
    ports:
      - '8080:80'
    depends_on:
      - redis
    restart: unless-stopped

  worker:
    image: fudaoyuanicu/015-worker:latest
    container_name: worker
    volumes:
      - ./uploads:/uploads
      - ./config.yaml:/config.yaml
    depends_on:
      - app
      - redis
    restart: unless-stopped

  redis:
    image: redis:7
    container_name: redis
    restart: unless-stopped
```

### Minimal `config.yaml`
```yaml
share:
  download_secret: "change-me-random-secret"
  download_window: 12
  password_salt: "change-me-random-salt"

upload:
  path: /uploads
  maximum: 100GiB

redis:
  url: redis://redis:6379/0

site:
  url: "https://your-domain.com"
  title:
    en: "015"
  desc:
    en: "Temporary file sharing"
```
Copy from `config.example.yaml` in the repo and edit before starting.

### Persistent volumes
| Path | Purpose |
|------|---------|
| `./uploads` (→ `/uploads`) | Uploaded files |
| `./config.yaml` (→ `/app/config.yaml`) | Application config |

### Features
- Large file chunked upload with frontend hash calculation
- Instant transfer (dedup via file hash + size)
- Image compression and format conversion
- File preview (images, video, audio, documents)
- Password-protected shares, download count limits, expiration times
- Pickup code system for simplified sharing
- Resume upload after interruption
- Chinese and English i18n
- JWT-based download token system

## Gotchas

- **`download_secret` and `password_salt` are required** — the app will not function correctly without them. Use long random strings.
- **`site.url` must match your public URL.** Share links are generated from this value; setting it to `localhost` will break links sent to others.
- **Files expire and are deleted automatically** after the configured window. There is no recovery.
- **Worker container processes async jobs** (image compression, etc.) — both `app` and `worker` must be running.
- **Config is in Chinese** in the example file — the keys are documented here in English for reference.
