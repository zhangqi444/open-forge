# ResourceSpace

Self-hosted Digital Asset Management (DAM) system for organising, managing, and sharing digital media (images, video, documents). PHP-based, MariaDB/MySQL backend, with transcoding and conversion via FFmpeg/ImageMagick. BSD-4-Clause. Upstream: https://www.resourcespace.com. Source (SVN/Git): https://www.resourcespace.com/svn. Docker repo: https://github.com/resourcespace/docker.

## Compatible install methods

| Method | When to use |
|---|---|
| Docker Compose (official image build) | Recommended; builds from the official Docker repo |
| Manual LAMP install | Existing Apache/PHP/MySQL server; full control |

## Software-layer concerns

| Concern | Detail |
|---|---|
| Language | PHP |
| Database | MariaDB or MySQL |
| Web server | Apache (in official Docker image) |
| Config file | include/config.php — generated by web setup wizard or pre-populated for Docker |
| Filestore | /var/www/html/filestore — all uploaded assets; must be persisted |
| Transcoding | FFmpeg + ImageMagick for automatic format conversion (installed in Docker image) |
| DB host in wizard | Enter "mariadb" (not "localhost") when using Docker Compose |
| MySQL binary path | Leave empty when using Docker Compose |

## Install: Docker Compose (official)

Source: https://www.resourcespace.com/knowledge-base/systemadmin/install_docker and https://github.com/resourcespace/docker

### 1. Install prerequisites

```bash
sudo apt update && sudo apt install git docker.io -y
# Follow Docker post-install steps: https://docs.docker.com/engine/install/linux-postinstall/
```

### 2. Clone the official Docker repository

```bash
git clone https://github.com/resourcespace/docker.git resourcespace
cd resourcespace
```

### 3. Configure credentials

Edit db.env — replace all "change-me" placeholders with secure passwords:

```bash
# db.env contents (example):
MYSQL_ROOT_PASSWORD=change-me
MYSQL_DATABASE=resourcespace
MYSQL_USER=resourcespace
MYSQL_PASSWORD=change-me
```

### 4. Build and start

```bash
docker compose up --build -d
```

### 5. Complete web setup wizard

Visit http://localhost (or your server IP). In the setup wizard:
- MySQL server: **mariadb** (not localhost)
- MySQL binary path: **leave empty**
- Enter your database name, user, and password from db.env

### docker-compose.yaml (reference)

```yaml
services:
  resourcespace:
    build: .
    container_name: resourcespace
    restart: unless-stopped
    depends_on:
      - mariadb
    volumes:
      - filestore:/var/www/html/filestore
      - ./config.php:/var/www/html/include/config.php
    networks:
      - frontend
      - backend
    ports:
      - "80:80"

  mariadb:
    image: mariadb
    container_name: mariadb
    restart: unless-stopped
    env_file:
      - db.env
    volumes:
      - mariadb:/var/lib/mysql
    networks:
      - backend

networks:
  frontend:
  backend:

volumes:
  mariadb:
  filestore:
```

## Install: Manual LAMP

Source: https://www.resourcespace.com/knowledge-base/systemadmin/install_overview

1. Check out or download source: https://www.resourcespace.com/svn
2. Set up Apache + PHP + MariaDB on your server
3. Place ResourceSpace files in your web root
4. Create a database and user
5. Make filestore/ writable by the web server: `chown -R www-data:www-data filestore/`
6. Visit the install URL to run the setup wizard
7. Configure FFmpeg and ImageMagick paths in the wizard for transcoding support

## Upgrade procedure

**Docker:**
```bash
cd resourcespace
git pull
docker compose up --build -d
```

**Manual:**
1. Back up the database and filestore/
2. Check out the new version from SVN/Git
3. Replace files (preserve include/config.php)
4. Run database migration if prompted in the UI

## Gotchas

- DB host must be "mariadb" in Docker: Using "localhost" in the setup wizard will fail — the container name is the hostname.
- MySQL binary path must be empty in Docker: ResourceSpace checks for the mysql binary; leave this blank in Docker setups.
- db.env passwords must be changed before build: The default "change-me" passwords are insecure. Change them before first `docker compose up`.
- config.php is mounted into the container: Editing docker-compose.yaml volumes ensures config.php survives container rebuilds.
- Filestore must be persisted: All uploaded files live in filestore/. If this volume is lost, uploaded assets are gone.
- Transcoding requires FFmpeg/ImageMagick: These are included in the Docker image. For manual installs, install and configure their paths.
