# MediaWiki

MediaWiki is the free, open-source wiki software that powers Wikipedia and thousands of other wikis. Written in PHP, it supports rich wikitext markup, extensions, templates, file uploads, and multi-language operation.

**Official site:** https://www.mediawiki.org  
**GitHub:** https://github.com/wikimedia/mediawiki  
**Upstream README:** https://github.com/wikimedia/mediawiki/blob/master/README.md  
**Docker Hub:** https://hub.docker.com/_/mediawiki  
**License:** GPL-2.0+

## Compatible Combos

| Infra | Runtime | Notes |
|-------|---------|-------|
| Any Linux VM / VPS | Docker Compose + MariaDB | Recommended |
| Any Linux VM / VPS | Docker Compose + PostgreSQL | Supported but less tested |
| Kubernetes | Bitnami Helm chart (`bitnami/mediawiki`) | Community supported |
| Bare metal | PHP + Apache/nginx + MySQL | Traditional LAMP stack |

## Inputs to Collect

### Before deployment
- Wiki name (e.g. `My Company Wiki`)
- Admin username and password (set via web installer)
- Database: MariaDB or MySQL (recommended) or PostgreSQL
- `MYSQL_DATABASE`, `MYSQL_USER`, `MYSQL_PASSWORD` for the database container
- Domain / URL for the wiki (important for image upload paths)
- `$wgSecretKey` — generated during web install (saved in `LocalSettings.php`)

### Optional
- SMTP settings for email notifications (password resets, watchlist emails)
- File upload storage path (local or S3 via Extension:AWS)
- Extensions to install (VisualEditor, CirrusSearch, etc.)

## Software-Layer Concerns

### Docker Compose

```yaml
version: "3.8"
services:
  mediawiki:
    image: mediawiki:latest
    container_name: mediawiki
    ports:
      - "8080:80"
    volumes:
      - mediawiki_images:/var/www/html/images
      - ./LocalSettings.php:/var/www/html/LocalSettings.php  # after initial setup
    depends_on:
      - db
    restart: unless-stopped

  db:
    image: mariadb:10.11
    container_name: mediawiki_db
    environment:
      MYSQL_DATABASE: mediawiki
      MYSQL_USER: wikiuser
      MYSQL_PASSWORD: ${MYSQL_PASSWORD:-changeme}
      MYSQL_RANDOM_ROOT_PASSWORD: "yes"
    volumes:
      - db_data:/var/lib/mysql
    restart: unless-stopped

volumes:
  mediawiki_images:
  db_data:
```

### First-run setup
1. Visit `http://your-server:8080/mw-config/` to run the web installer
2. Complete the setup wizard; download the generated `LocalSettings.php`
3. Place `LocalSettings.php` in the wiki root (mount it into the container at `/var/www/html/LocalSettings.php`)
4. Restart the container

### Key paths
| Path | Purpose |
|------|---------|
| `/var/www/html/LocalSettings.php` | Main config file (generated by installer) |
| `/var/www/html/images/` | Uploaded files — mount as persistent volume |
| `/var/www/html/extensions/` | Extension directory |

### Ports
| Port | Purpose |
|------|---------|
| 80 (container) | HTTP — proxy with nginx/Caddy |

### Important `LocalSettings.php` variables
```php
$wgServer = "https://wiki.example.com";
$wgEnableUploads = true;
$wgAllowExternalImages = false;
$wgDBserver = "db";
$wgDBname = "mediawiki";
$wgDBuser = "wikiuser";
$wgDBpassword = "changeme";
```

## Upgrade Procedure

1. Back up database: `mysqldump -u wikiuser -p mediawiki > wiki_backup.sql`
2. Back up the `images` volume and `LocalSettings.php`
3. Pull new image: `docker pull mediawiki:latest`
4. Restart: `docker compose down && docker compose up -d`
5. Run DB migrations: `docker exec -it mediawiki php /var/www/html/maintenance/update.php`
6. Clear parser cache if needed: `php maintenance/rebuildall.php`

---

## Gotchas

- **`LocalSettings.php` is required** — without it MediaWiki shows the installer on every request; after completing setup, mount the file into the container
- **Images volume is critical** — back it up alongside the database; uploaded files are stored here, not in the DB
- **`update.php` after upgrades** — MediaWiki does not auto-run DB migrations; run `maintenance/update.php` manually after each version upgrade
- **Extension compatibility** — extensions must match the MediaWiki version; check extension compatibility tables before upgrading
- **`$wgServer` must match the actual URL** — mismatches cause broken image links and OAuth issues
- **File upload permissions** — the `images/` directory must be writable by the web server (www-data); in Docker this is handled automatically
- **Search** — built-in search is basic; for full-text search install CirrusSearch (backed by Elasticsearch)
