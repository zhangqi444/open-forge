# Redash

Open-source data querying, visualization, and dashboard platform. Supports 35+ SQL and NoSQL data sources. Upstream: <https://github.com/getredash/redash>. Setup guide: <https://redash.io/help/open-source/setup>.

Redash runs as a Python (Flask/Celery) web application backed by PostgreSQL and Redis. The server process handles the web UI and API; multiple worker processes handle query execution, scheduling, and email.

## Software-layer concerns

### Environment file (`/opt/redash/env`)

The setup script generates `/opt/redash/env`. Key variables:

```bash
PYTHONUNBUFFERED=0
REDASH_LOG_LEVEL=INFO
REDASH_REDIS_URL=redis://redis:6379/0
POSTGRES_PASSWORD=<auto-generated>
REDASH_COOKIE_SECRET=<auto-generated>
REDASH_SECRET_KEY=<auto-generated>
REDASH_DATABASE_URL=postgresql://postgres:<POSTGRES_PASSWORD>@postgres/postgres
# Optional SMTP:
REDASH_MAIL_SERVER=smtp.example.com
REDASH_MAIL_PORT=587
REDASH_MAIL_USE_TLS=true
REDASH_MAIL_USERNAME=user@example.com
REDASH_MAIL_PASSWORD=secret
REDASH_MAIL_DEFAULT_SENDER=redash@example.com
# Public URL:
REDASH_HOST=https://redash.example.com
```

### Services (Docker Compose)

From `getredash/setup` `data/compose.yaml`:

| Service | Role |
|---|---|
| `server` | Web + API (`port 5000`), `REDASH_WEB_WORKERS=4` |
| `scheduler` | Cron-style scheduled query runner |
| `scheduled_worker` | Executes scheduled queries and schema refreshes |
| `adhoc_worker` | Executes ad-hoc user queries |
| `worker` | Handles periodic jobs, emails, default queue |
| `redis` | Queue backend (`redis:7-alpine`) |
| `postgres` | Metadata store (`pgautoupgrade:17-alpine`) |
| `nginx` | Reverse proxy on port `80` |

### Data directories

| Path | Contents |
|---|---|
| `/opt/redash/postgres-data` | PostgreSQL data volume |
| `/opt/redash/env` | Environment file with secrets |

### Install steps (Docker Compose on Ubuntu)

Based on <https://redash.io/help/open-source/setup> and <https://github.com/getredash/setup>:

```bash
# 1. Install Docker Engine
curl -fsSL https://get.docker.com | bash

# 2. Download and run the setup script
wget https://raw.githubusercontent.com/getredash/setup/master/setup.sh
chmod +x setup.sh
sudo ./setup.sh

# The setup script:
# - Installs Docker Compose
# - Creates /opt/redash/ directory structure
# - Generates /opt/redash/env with random secrets
# - Downloads compose.yaml to /opt/redash/
# - Runs: docker compose run --rm server create_db
# - Starts all services via docker compose up -d

# 3. Access Redash at http://<server-ip>
# 4. Complete setup wizard in browser (set admin email/password)
```

### First-time database initialization

The setup script runs this automatically, but if doing it manually:

```bash
cd /opt/redash
docker compose run --rm server create_db
docker compose up -d
```

## Upgrade procedure

From <https://redash.io/help/open-source/maintenance-and-operations/how-to-upgrade-redash>:

```bash
cd /opt/redash

# 1. Edit compose.yaml — update the image tag in x-redash-service anchor
# Change: image: redash/redash:__TAG__
# To:     image: redash/redash:<new-version>

# 2. Pull new images
docker compose pull

# 3. Run any pending database migrations
docker compose run --rm server manage db upgrade

# 4. Restart services
docker compose up -d
```

⚠️ Always check the [release notes](https://github.com/getredash/redash/releases) for breaking changes before upgrading.

## Reverse proxy / TLS

The default Docker Compose exposes Nginx on port 80 only. For HTTPS:

1. Place Nginx or Caddy in front with TLS termination, or
2. Modify the nginx service in `compose.yaml` to add a `443` port and mount TLS certificates.

Set `REDASH_HOST=https://your.domain.com` in `/opt/redash/env` so Redash generates correct absolute URLs for email links.

## Gotchas

- **PostgreSQL version pin**: The compose file intentionally pins `pgautoupgrade:17-alpine` to avoid auto-upgrade to v18+. Do not change this without following the [pgautoupgrade upgrade guide](https://github.com/pgautoupgrade/docker-pgautoupgrade#one-shot-mode).
- **`REDASH_HOST` must be set**: Without this, email alert links and invite links will use `localhost`.
- **Workers restart**: After updating the env file, restart ALL services (`docker compose down && docker compose up -d`), not just the server.
- **Schema refresh**: If connected database schema doesn't appear, trigger it via Settings → Data Sources → Refresh Schema.
- **Rate limiting for alerts**: Alerts use the worker queue. If the `worker` container is not running, alerts will not fire.
- **Redis persistence**: By default Redis uses no persistence. If Redis restarts, pending query jobs are lost (they re-queue on next poll).
- **Secrets rotation**: Changing `REDASH_COOKIE_SECRET` or `REDASH_SECRET_KEY` invalidates all existing sessions and API keys respectively.
