# F-Droid Server (fdroidserver)

Suite of command-line tools to create and maintain your own F-Droid Android app repository. You can publish APKs you've already built, or use the full build server to compile Android apps from source using the Android SDK. Powers the main f-droid.org repository. Written in Python.

Upstream: <https://gitlab.com/fdroid/fdroidserver> | Docs: <https://f-droid.org/docs>

## Compatible combos

| Infra | Runtime | Notes |
|---|---|---|
| Debian/Ubuntu | apt (fdroidserver package) | Stable release; may be outdated — use PPA for newer |
| Ubuntu/Mint | F-Droid PPA | Newer release than standard apt |
| Debian | backports | `apt-get install -t bookworm-backports fdroidserver` |
| Fedora/Arch | pip in virtualenv | Recommended to avoid dependency conflicts |

## Software-layer concerns

### What fdroidserver does NOT include

fdroidserver is a toolset, not a web server. It:
1. Manages a directory of APKs + metadata
2. Signs packages with your key
3. Generates repo index files (index.jar, index-v2.jar)
4. You then serve that directory via any web server (nginx, Apache, S3, etc.)

### Config file

`config.yml` lives in the repo directory. Key settings:

```yaml
# Repo identity
repo_url: https://myfdrep.example.com/fdroid/repo
repo_name: My Personal Repo
repo_description: My personal F-Droid repository
repo_icon: icon.png

# Signing — generated by fdroid init; keep keystore private
keystorepass: changeme
keystore: keystore.p12
repo_keyalias: repo

# Optional: build server settings
build_server_always: false
```

### Data dirs

- `repo/` — signed APKs and generated index files (serve this directory)
- `metadata/` — YAML metadata files per app (optional, for build-from-source mode)
- `keystore.p12` — signing keystore (keep private, back up)
- `config.yml` — repo configuration

## Install

```bash
# Option 1: Debian/Ubuntu (apt)
sudo apt-get install fdroidserver

# Option 2: Ubuntu PPA (newer version)
sudo add-apt-repository ppa:fdroid/fdroidserver
sudo apt-get update
sudo apt-get install fdroidserver

# Option 3: Debian backports
sudo apt-get install -t bookworm-backports fdroidserver

# Option 4: pip in virtualenv (any platform)
python3 -m venv fdroidserver-env
source fdroidserver-env/bin/activate
pip install fdroidserver
# Or from source:
pip install git+https://gitlab.com/fdroid/fdroidserver.git

# Option 5: Homebrew (macOS)
brew install fdroidserver

# Option 6: Guix
guix install fdroidserver
```

## Set up a repo

```bash
mkdir myrepo && cd myrepo

# Initialize: generates config.yml and signing keystore
fdroid init

# Edit config.yml with your repo URL, name, description

# Add APKs to the repo/ directory
cp /path/to/app.apk repo/

# Update: signs APKs, generates index
fdroid update --create-metadata

# Verify
fdroid readmeta

# Serve the repo/ directory with nginx/Apache:
# - Document root: /path/to/myrepo
# - Users add: https://yourserver/fdroid/repo  as an F-Droid repo URL
```

## Automate updates via cron

```bash
# Sign and publish whenever new APKs are added
crontab -e
# Add:
0 * * * * cd /path/to/myrepo && fdroid update --create-metadata >> /var/log/fdroid-update.log 2>&1
```

## Upgrade procedure

```bash
# apt
sudo apt-get upgrade fdroidserver

# pip
pip install --upgrade fdroidserver

# Docker: pull new image
docker pull registry.gitlab.com/fdroid/fdroidserver:latest
```

## Gotchas

- The repo/ directory must be served over HTTPS for F-Droid clients to accept it — HTTP is blocked by default in modern F-Droid.
- `fdroid init` generates a new keystore — keep keystore.p12 and your keystorepass backed up. Losing the signing key means clients will see your repo as a different (untrusted) repo on the next update.
- fdroidserver in standard Debian/Ubuntu repos is often out of date — use the PPA or pip for the latest version.
- Building apps from source requires the full Android SDK and Java — see [Build Server Setup](https://f-droid.org/docs/Build_Server_Setup/) docs. For just publishing pre-built APKs, no SDK is needed.
- Docker executable image is useful for CI pipelines (signing + index generation) without installing Python deps.
