# phpBB

Free, open-source PHP bulletin board (forum) software. Long-standing community platform with extensions, styles, and a large plugin ecosystem. Upstream: <https://github.com/phpbb/phpbb>. Official site: <https://www.phpbb.com>.

**License:** GPL-2.0

## Compatible install methods

| Method | Upstream | First-party? | When to use |
|---|---|---|---|
| Traditional PHP web server (LAMP/LEMP) | https://www.phpbb.com/support/docs/en/3.3/ug/quickstart/ | ✅ | Standard deployment; shared hosting or VPS with PHP stack |
| Docker (bitnami/phpbb) | https://hub.docker.com/r/bitnami/phpbb | Community (Bitnami) | Containerised deployment with MariaDB |
| Vagrant (dev only) | https://github.com/phpbb/phpbb/blob/master/phpBB/docs/vagrant.md | ✅ | Development and contribution only |

## Inputs to collect

| Phase | Prompt | Format | Applicability |
|---|---|---|---|
| database | "Database type?" | MySQL/MariaDB or PostgreSQL | All |
| database | "Database host/name/user/password?" | Free-text | All |
| app | "Forum domain?" | e.g. forum.example.com | All |
| app | "Admin username and password?" | Free-text | All |
| email | "SMTP settings?" | host/port/user/pass | Optional; needed for registration emails |

## Docker Compose (Bitnami)

Bitnami provides the most maintained Docker image for phpBB:

```yaml
services:
  mariadb:
    image: bitnami/mariadb:latest
    environment:
      - MARIADB_ROOT_PASSWORD=rootpassword
      - MARIADB_DATABASE=phpbb
      - MARIADB_USER=phpbb
      - MARIADB_PASSWORD=phpbbpassword
    volumes:
      - mariadb-data:/bitnami/mariadb
    restart: unless-stopped

  phpbb:
    image: bitnami/phpbb:latest
    ports:
      - 80:8080
      - 443:8443
    environment:
      - PHPBB_DATABASE_HOST=mariadb
      - PHPBB_DATABASE_PORT_NUMBER=3306
      - PHPBB_DATABASE_USER=phpbb
      - PHPBB_DATABASE_PASSWORD=phpbbpassword
      - PHPBB_DATABASE_NAME=phpbb
      - PHPBB_USERNAME=admin
      - PHPBB_PASSWORD=adminpassword
      - PHPBB_EMAIL=admin@example.com
      - PHPBB_FORUM_NAME=My Forum
      - PHPBB_SMTP_HOST=smtp.example.com
      - PHPBB_SMTP_PORT=587
      - PHPBB_SMTP_USER=user@example.com
      - PHPBB_SMTP_PASSWORD=smtppassword
    volumes:
      - phpbb-data:/bitnami/phpbb
    depends_on:
      - mariadb
    restart: unless-stopped

volumes:
  mariadb-data:
  phpbb-data:
```

## Traditional PHP install

```bash
# Download from https://www.phpbb.com/downloads/
# Extract to web root
unzip phpBB-<version>.zip -d /var/www/html/forum

# Set permissions
chown -R www-data:www-data /var/www/html/forum
chmod -R 755 /var/www/html/forum

# Visit http://yourdomain.com/forum/install/ in browser
# Follow web installer: database, admin account, email settings

# After install, delete or protect the /install directory
rm -rf /var/www/html/forum/install
```

## Software-layer concerns

### PHP requirements

- PHP 7.4+ (8.x recommended for phpBB 3.3.x)
- Extensions: mysqli/pgsql, gd, mbstring, xml, zip, zlib, curl, intl
- MySQL 5.5+ / MariaDB 10.1+ / PostgreSQL 9.x+

### Key config files

| File | Purpose |
|---|---|
| `config.php` (generated by installer) | Database connection, core settings |
| `store/` | Cached data; must be writable |
| `files/` | User-uploaded attachments; must be writable |
| `images/avatars/upload/` | User avatars; must be writable |

### Admin Control Panel (ACP)

All forum settings (permissions, styles, extensions, email) are managed via the ACP at `/adm/index.php`.

## Upgrade procedure

```bash
# Official upgrade guide: https://www.phpbb.com/support/docs/en/3.3/ug/upgrade/

# 1. Back up files and database
# 2. Download new version package
# 3. Upload files (do NOT overwrite config.php)
# 4. Visit /install/app.php?mode=update in browser
# 5. Follow prompts; delete /install/ when done
```

## Gotchas

- **Delete /install after setup.** The web installer must be removed or blocked after installation; leaving it accessible is a critical security risk.
- **config.php is write-protected.** The installer generates `config.php`. Do not overwrite it during upgrades.
- **File permissions are critical.** `store/`, `files/`, and `images/avatars/upload/` must be writable by the web server user.
- **Extensions and styles must be compatible.** After a major phpBB version upgrade, extensions and custom styles may need updating. Check the Extension Database (https://www.phpbb.com/customise/db/) for version compatibility.
- **Bitnami image vs upstream.** The upstream phpbb/phpbb repo does not ship an official Docker image. The Bitnami image is community-maintained but well-supported and widely used.
- **No built-in HTTPS.** phpBB serves over HTTP; put it behind a reverse proxy (nginx/Caddy/Traefik) for TLS termination.
