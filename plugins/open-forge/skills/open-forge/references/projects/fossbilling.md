# FOSSBilling

Free, open-source billing and client management system for online service businesses. Automates invoicing, payments, and client communication. Popular with web hosting providers; also suitable for software licensing, digital downloads, and subscriptions. Upstream: <https://github.com/FOSSBilling/FOSSBilling>. Docs: <https://fossbilling.org/docs>.

**License:** Apache-2.0

> ⚠️ **FOSSBilling is currently in beta.** Expect rough edges and test thoroughly before using in production.

## Install (release ZIP — recommended)

```bash
# Download latest release from https://github.com/FOSSBilling/FOSSBilling/releases/latest
wget https://github.com/FOSSBilling/FOSSBilling/releases/latest/download/FOSSBilling.zip
unzip FOSSBilling.zip -d /var/www/html/billing

# Set permissions
chown -R www-data:www-data /var/www/html/billing
chmod -R 755 /var/www/html/billing
chmod -R 775 /var/www/html/billing/data

# Configure nginx/Apache to serve /var/www/html/billing/
# Visit https://billing.example.com/install/ to run the web installer
```

After the installer completes, **delete the `install/` directory**:
```bash
rm -rf /var/www/html/billing/install
```

## Docker

```yaml
services:
  fossbilling:
    image: fossbilling/fossbilling:latest
    restart: always
    ports:
      - 80:80
    volumes:
      - ./data:/var/www/html/data
    environment:
      - DB_HOST=mysql
      - DB_NAME=fossbilling
      - DB_USER=fossbilling
      - DB_PASS=password

  mysql:
    image: mysql:8.0
    restart: always
    environment:
      - MYSQL_DATABASE=fossbilling
      - MYSQL_USER=fossbilling
      - MYSQL_PASSWORD=password
      - MYSQL_ROOT_PASSWORD=rootpassword
    volumes:
      - ./mysql:/var/lib/mysql
```

## Software-layer concerns

### PHP requirements

Check [fossbilling.org/docs/getting-started/requirements](https://fossbilling.org/docs/getting-started/requirements) for the current PHP version and extension requirements (changes between releases).

### Key directories

| Directory | Purpose |
|---|---|
| `data/` | Config files, caches, logs, uploads; must be writable |
| `data/cache/` | Template/data cache |
| `data/uploads/` | Client-uploaded files |
| `config.php` (in data/) | Generated by installer; DB credentials and app settings |

### Cron job

Required for automated invoicing, reminder emails, and task processing:

```cron
*/5 * * * * php /var/www/html/billing/cron.php
```

## Upgrade procedure

```bash
# Official: https://fossbilling.org/docs/getting-started/updating-fossbilling
# 1. Back up database and data/ directory
# 2. Download new release, extract (preserve data/config.php)
# 3. Visit https://billing.example.com/install/ — upgrade mode runs if update is detected
# 4. Delete install/ when done
```

## Gotchas

- **Beta status.** FOSSBilling is actively developed and considered beta; verify features you need work before deploying in production.
- **Delete install/ after setup.** Leaving the installer accessible is a security vulnerability.
- **Cron job is required.** Without cron, invoices won't be sent automatically, trial periods won't expire, etc.
- **data/ must be writable** but should not be web-accessible. Configure nginx/Apache to deny access to `data/`.
- **config.php is generated.** Do not overwrite it during upgrades.
- **FOSSBilling is a BoxBilling fork.** BoxBilling is abandoned; FOSSBilling is the maintained successor.
