# Bluesky PDS

Official Personal Data Server for the Bluesky social network, built on AT Protocol. Self-hosting gives you a custom domain handle (e.g. `@you.example.com`), full ownership of your posts/follows/likes, and federation with the wider Bluesky network. Upstream: <https://github.com/bluesky-social/pds>. Install docs: <https://github.com/bluesky-social/pds/blob/main/README.md>.

The PDS stores your repo (posts, follows, likes, DMs) and federates with the AT Protocol relay network. A bundled Caddy container handles TLS automatically; Watchtower auto-updates the image.

## Compatible combos

| Infra | Runtime | Notes |
|---|---|---|
| Any VPS (public IPv4) | Docker Compose | Official method. Ubuntu 20.04/22.04/24.04 or Debian 11/12/13. |
| Bare metal (public IPv4) | Docker Compose | Same requirements — public IP and public DNS name required. |

> ℹ️ Cannot run behind a NAT or private network without a public IP. Ports 80/tcp and 443/tcp must be reachable from the internet.

## Inputs to collect

| Phase | Prompt | Notes |
|---|---|---|
| dns | "What public domain will your PDS be at (e.g. `pds.example.com`)?" | A-record must point to the server before running the installer. |
| admin | "Admin email address?" | Used for Let's Encrypt TLS cert. |
| admin | "Admin password for the PDS?" | Used to create the first account via `pdsadmin`. |
| smtp (optional) | "SMTP credentials for email notifications?" | Optional — PDS works without SMTP but can't send email notifications. |

## Software-layer concerns

### Config paths
All PDS config and data lives under `/pds/`:
- `/pds/pds.env` — environment configuration file (generated by installer or created manually)
- `/pds/pdsadmin.sh` — management script
- `/pds/caddy/` — Caddy TLS data
- `/pds/blocks/` — block storage for repo data

### Key env vars (in `/pds/pds.env`)

```
PDS_HOSTNAME=pds.example.com
PDS_JWT_SECRET=<random>
PDS_ADMIN_PASSWORD=<your-admin-password>
PDS_PLC_ROTATION_KEY_K256_PRIVATE_KEY_HEX=<generated>
PDS_DATA_DIRECTORY=/pds
PDS_BLOBSTORE_DISK_LOCATION=/pds/blocks
PDS_DID_PLC_URL=https://plc.directory
PDS_BSKY_APP_VIEW_URL=https://api.bsky.app
PDS_BSKY_APP_VIEW_DID=did:web:api.bsky.app
PDS_REPORT_SERVICE_URL=https://mod.bsky.app
PDS_REPORT_SERVICE_DID=did:plc:ar7c4by46qjdydhdevvrndac
PDS_CRAWLERS=https://bsky.network
LOG_ENABLED=true

# Optional SMTP
PDS_EMAIL_SMTP_URL=smtps://user:password@smtp.host:465/
PDS_EMAIL_FROM_ADDRESS=noreply@pds.example.com
```

### Docker Compose (from upstream)

```yaml
version: '3.9'
services:
  caddy:
    container_name: caddy
    image: caddy:2
    network_mode: host
    depends_on:
      - pds
    restart: unless-stopped
    volumes:
      - type: bind
        source: /pds/caddy/data
        target: /data
      - type: bind
        source: /pds/caddy/etc/caddy
        target: /etc/caddy
  pds:
    container_name: pds
    image: ghcr.io/bluesky-social/pds:0.4
    network_mode: host
    restart: unless-stopped
    volumes:
      - type: bind
        source: /pds
        target: /pds
    env_file:
      - /pds/pds.env
  watchtower:
    container_name: watchtower
    image: ghcr.io/nicholas-fedor/watchtower:latest
    network_mode: host
    volumes:
      - type: bind
        source: /var/run/docker.sock
        target: /var/run/docker.sock
    restart: unless-stopped
    environment:
      WATCHTOWER_CLEANUP: true
      WATCHTOWER_SCHEDULE: "@midnight"
```

> Note: `network_mode: host` is used by all services for simplicity and port sharing.

### Ports
- 80/tcp — ACME challenge (Caddy, auto-redirects to HTTPS)
- 443/tcp — HTTPS (all API traffic)

## Installation

### Using the official installer script (recommended)

```bash
# As root on Ubuntu/Debian
curl -fsSL https://raw.githubusercontent.com/bluesky-social/pds/main/installer.sh | bash
```

The installer:
1. Creates `/pds/` directory structure
2. Generates all secrets automatically
3. Writes `/pds/pds.env` and `docker-compose.yml`
4. Creates `/pds/caddy/etc/caddy/Caddyfile`
5. Starts the stack

### Manual setup

1. Create `/pds/` and subdirectories
2. Copy `compose.yaml` from upstream repo
3. Create `/pds/pds.env` with required vars (see above)
4. Create Caddyfile at `/pds/caddy/etc/caddy/Caddyfile`
5. `docker compose -f /pds/compose.yaml up -d`

### Account management (`pdsadmin`)

```bash
# Create an account
/pds/pdsadmin.sh account create

# Create invite code
/pds/pdsadmin.sh generate-invite-code

# List accounts
/pds/pdsadmin.sh account list

# Request crawler re-index (after PDS restore/migration)
/pds/pdsadmin.sh request-crawl
```

### goat CLI (bundled in container)

The PDS container also ships `goat` — a lower-level AT Protocol admin CLI. Run commands via `docker exec`:

```bash
# Create an account (alternative to pdsadmin)
docker exec pds goat pds admin account create --admin-password YOUR_ADMIN_PASS \
  --handle user.pds.example.com --email user@email.com --password userpass

# Create invite codes
docker exec pds goat pds admin create-invites --admin-password YOUR_ADMIN_PASS

# Check relay status
docker exec pds sh -c 'goat relay host status "$PDS_HOSTNAME" --json'
```

## Upgrade procedure

Watchtower handles automatic nightly updates. To update manually:

```bash
docker compose -f /pds/compose.yaml pull
docker compose -f /pds/compose.yaml up -d
```

Check release notes at: <https://github.com/bluesky-social/pds/releases>

## Gotchas

- **Public IP required** — PDS must be reachable from the internet. NAT/private-IP setups won't federate.
- **DNS must resolve before TLS** — Caddy requests a Let's Encrypt cert on startup; if DNS isn't propagated, the PDS will start but TLS will fail.
- **`network_mode: host`** — All three containers (Caddy, PDS, Watchtower) use host networking. No ports are published via Docker; they bind directly to the host's network interfaces.
- **Handle migration** — If you want your Bluesky handle to be `@user.example.com` instead of `@user.pds.example.com`, configure a `_atproto` TXT DNS record or a `/.well-known/atproto-did` route at the apex domain.
- **Relay desync** — After restoring from backup or migrating, run `pdsadmin request-crawl` to force the relay to re-index your PDS.
- **ARM support** — The official image supports `amd64` and `arm64`; runs on Raspberry Pi 4/5.
- **No built-in web UI** — The PDS is an API server only. Use the official Bluesky app (`bsky.app`) or a third-party client; point it to your PDS host during sign-in.
