# Pydio Cells

Enterprise-grade file sharing and collaboration platform. Secure file sync, sharing, and team collaboration with fine-grained access control, audit logs, S3 storage support, and a micro-service architecture. Full rewrite of the original Pydio project in Go.

2,200 stars · AGPL-3.0

Website: https://pydio.com/
Docs: https://pydio.com/en/docs
Demo: https://pydio.com/en/demo
Docker Hub: https://hub.docker.com/r/pydio/cells

## What it is

Pydio Cells provides a full enterprise file management platform:

- **File sync** — Desktop clients (Mac/Windows/Linux) for two-way sync
- **File sharing** — Share files/folders internally and via public links
- **Fine-grained permissions** — Workspace-based ACLs: read, write, deny per user/group
- **Workspaces** — Organize files into logical units with independent access control
- **Versioning** — File version history and restore
- **Metadata** — Custom metadata fields per file
- **Full-text search** — Search file contents and metadata
- **Preview** — Image, video, PDF, and office document preview in browser
- **Micro-service architecture** — Split into specialized services for scalability
- **S3 compatible storage** — Use local filesystem, S3, Minio, Azure Blob as backends
- **Audit logs** — Complete audit trail of all file operations
- **OpenID Connect** — SSO integration with external identity providers
- **REST API** — Full API for custom integrations
- **Cells Flows** — Visual workflow automation (paid Cells Enterprise)
- **Team collaboration** — Cell-based workspaces for teams with chat, comments

## Inputs to collect

### Phase 1 — Pre-install
- Domain name and HTTPS setup (required for sync clients)
- Database: MySQL 5.7+/8.0+ or PostgreSQL 11+
- Storage path for files
- Admin email and password
- External URL (used in sharing links and sync config)

## Software-layer concerns

### Ports
- 8080 — HTTP (use reverse proxy for HTTPS)
- 33060 — gRPC (internal micro-services; do not expose publicly)

### Config paths
- /var/cells/ — application data, configs, and local file storage
- ~/.config/pydio/cells/ (bare metal) — config directory
- pydio.json — main application config (auto-generated by installer)

### Docker quick start (with embedded SQLite for dev)
  docker run -d --name cells \
    -p 8080:8080 \
    -v cells_data:/var/cells \
    pydio/cells:4.4.17

Access at http://localhost:8080 — follow the web installer.

### Docker Compose (production with MySQL)
  version: '3'
  services:
    cells:
      image: pydio/cells:4.4.17
      restart: unless-stopped
      ports:
        - "8080:8080"
      environment:
        - CELLS_SITE_BIND=0.0.0.0:8080
        - CELLS_SITE_EXTERNAL=https://files.example.com
        - CELLS_SITE_NO_TLS=1
      volumes:
        - cells_data:/var/cells
      depends_on:
        - mysql

    mysql:
      image: mysql:8.0
      restart: unless-stopped
      environment:
        MYSQL_ROOT_PASSWORD: rootpass
        MYSQL_DATABASE: cells
        MYSQL_USER: cells
        MYSQL_PASSWORD: cellspass
      volumes:
        - mysqldata:/var/lib/mysql

  volumes:
    cells_data:
    mysqldata:

First run visits http://localhost:8080 for the web installer (configure DB, admin user, storage).

### Reverse proxy (Nginx)
  server {
    listen 443 ssl;
    server_name files.example.com;
    client_max_body_size 0;
    location / {
      proxy_pass http://localhost:8080;
      proxy_http_version 1.1;
      proxy_set_header Upgrade $http_upgrade;
      proxy_set_header Connection "upgrade";
      proxy_set_header Host $host;
      proxy_set_header X-Forwarded-Proto https;
    }
  }

## Upgrade procedure

1. Backup database: mysqldump -u cells -p cells > backup.sql
2. Backup /var/cells/data/ (file storage)
3. docker pull pydio/cells:4.4.17
4. docker compose up -d --force-recreate cells
5. Cells applies database migrations automatically on start
6. Verify sync clients reconnect and file access works

## Gotchas

- Web installer on first run — visit HTTP port after first start to configure DB and admin account; do not skip this step
- HTTPS for sync clients — desktop sync clients require HTTPS; use a reverse proxy with Let's Encrypt
- WebSocket needed — upgrade proxy headers required for the real-time features; ensure Nginx passes Upgrade headers
- client_max_body_size 0 — required in Nginx for large file uploads; default Nginx limit will break uploads
- gRPC port — port 33060 is for internal micro-service communication; do not expose to the internet
- AGPL-3.0 — public deployments with modifications must release source; enterprise features (Cells Flows, etc.) require a commercial Cells Enterprise license
- Cells Enterprise vs Community — the Docker image is Community edition; Enterprise adds advanced workflows, office integration, and enterprise SSO
